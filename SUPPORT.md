# Support

This page explains how to get help for repositories in the ReadLab X
organization when a repository does not provide its own support guidance.

## Where to ask for help

Use the path that best matches your question:

- Use GitHub issues for bugs and reproducible problems.
- Use pull requests for code or content changes you are ready to propose.
- Use email for private questions or security-sensitive matters.

## Contact

- GitHub organization: <https://github.com/readlab-x>
- Email: <mailto:6iedog@gmail.com>

## What to include

If you are asking for help on a repository, include:

- the repository name
- the page, feature, or file involved
- what you expected
- what happened instead
- screenshots or logs when relevant

## Response expectations

Support is best-effort. Response times may vary depending on project status,
maintenance capacity, and whether the request is public, private, or
security-related.
