# Security policy

This document explains how to report security issues for repositories under
the ReadLab X organization.

## Reporting a vulnerability

If you find a security issue, report it privately instead of opening a public
issue.

- Email: <mailto:6iedog@gmail.com>
- Subject line: `Security report for ReadLab X`

Include the following details when possible:

- the affected repository and branch
- a clear description of the issue
- steps to reproduce the problem
- impact and attack scenario
- suggested remediation, if you have one

## What to expect

After you send a report, you can expect the following:

1. We review the report and confirm receipt.
2. We assess severity and scope.
3. We work on mitigation or a fix.
4. We coordinate disclosure when appropriate.

Do not publish proof-of-concept details before the issue is acknowledged and
the maintainers have had a reasonable chance to respond.

## Scope

This policy applies to repositories owned by the ReadLab X GitHub
organization unless a repository provides its own `SECURITY.md`.
